# 模块答辩术语集

## 第二模块：缓存模块

### 模块定位
缓存模块是系统性能优化的核心组件，负责管理系统各类数据的缓存策略，提升数据访问效率，减轻数据库压力，确保系统在高并发场景下的响应速度和稳定性。

### 核心技术
- **Caffeine缓存框架**：高性能本地缓存库，提供低延迟的数据访问能力
- **Spring Cache抽象层**：简化缓存操作，实现缓存与业务逻辑解耦
- **缓存预热机制**：在应用启动时预加载关键数据，避免冷启动性能问题
- **多级缓存策略**：针对不同数据类型设计差异化的缓存配置

### 关键实现
1. **缓存配置设计**：通过`CacheConfig.java`定义多个缓存区域（categories、books、stocks等），为每个区域配置独立的大小限制、过期时间和淘汰策略
2. **权重化缓存管理**：对分类数据采用基于权重的缓存淘汰策略，优先保留热门分类
3. **缓存预热机制**：通过`CacheWarmer.java`在应用启动时预加载分类列表、热门图书等关键数据
4. **缓存失效策略**：实现数据更新时的缓存自动失效，确保数据一致性
5. **分层缓存应用**：在控制器层和服务层通过`@Cacheable`、`@CachePut`等注解实现缓存的精细化控制

### 价值体现
- **性能提升**：将热点数据缓存到内存，查询响应时间从毫秒级提升到微秒级
- **系统稳定性**：降低数据库访问压力，避免高并发场景下的数据库瓶颈
- **用户体验优化**：页面加载速度显著提升，减少用户等待时间
- **资源利用效率**：通过合理的缓存配置和淘汰策略，平衡内存使用与缓存命中率

## 第三模块：订单管理模块

### 模块定位
订单管理模块是系统的核心业务组件，负责处理用户订单的全生命周期管理，包括订单创建、支付处理、订单状态更新、订单查询等功能，是连接用户、商品和支付系统的关键桥梁。

### 核心技术
- **MyBatis持久层框架**：实现订单数据的高效访问和操作
- **Spring事务管理**：确保订单创建、库存扣减等操作的原子性
- **分层架构设计**：遵循Controller-Service-Mapper分层模式，实现业务逻辑解耦
- **RESTful API设计**：提供标准化的订单操作接口

### 关键实现
1. **订单数据模型**：设计完整的订单实体结构，支持订单基本信息、商品明细、支付信息等数据的存储和管理
2. **订单创建流程**：实现从购物车到订单创建的完整流程，包括库存检查、价格计算、订单号生成等环节
3. **事务管理机制**：在订单创建过程中使用声明式事务，确保订单创建、库存扣减、日志记录等操作的原子性
4. **订单状态流转**：设计清晰的订单状态机，支持订单从创建、支付、发货到完成的全状态流转
5. **后台订单管理**：提供订单查询、状态修改、统计分析等后台管理功能，支持按条件筛选和批量操作

### 价值体现
- **业务完整性**：实现了电商系统的核心订单功能，支撑完整的用户购物流程
- **数据一致性**：通过事务管理确保订单数据与库存、用户账户等数据的一致性
- **操作便捷性**：提供友好的订单管理界面，方便管理员进行订单处理和统计分析
- **扩展性**：模块化设计支持未来订单功能的扩展，如订单拆分、合并等高级功能

## 第四模块：安全模块

### 模块定位
安全模块是系统的防护屏障，负责保障系统的数据安全、用户隐私和操作安全，通过多层次的安全机制抵御各类安全威胁，确保系统的稳定性和可信度。

### 核心技术
- **Spring Security框架**：提供全面的安全认证和授权机制
- **自定义过滤器链**：实现CSRF防护和XSS攻击过滤
- **RBAC权限模型**：基于角色的访问控制，实现精细化的权限管理
- **安全配置管理**：集中管理系统安全策略和配置

### 关键实现
1. **Spring Security配置**：通过`SecurityConfig.java`配置系统安全策略，包括资源访问规则、认证机制等
2. **CSRF防护机制**：实现`CSRFFilter`过滤器，对所有状态变更请求进行CSRF令牌验证
3. **XSS攻击过滤**：实现`XSSFilter`过滤器，对用户输入进行安全过滤和转义
4. **权限控制**：结合Spring Security和自定义拦截器，实现基于角色的访问控制
5. **安全测试**：编写`SecurityFiltersTest.java`测试类，验证安全过滤器的有效性

### 价值体现
- **安全防护**：有效抵御CSRF、XSS等常见Web安全攻击
- **数据保护**：确保用户敏感数据的安全存储和传输
- **合规性**：满足Web应用安全开发规范，提升系统可信度
- **用户信任**：为用户提供安全的操作环境，增强用户对系统的信任
